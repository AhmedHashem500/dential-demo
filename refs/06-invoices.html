Product: Dential Desktop · Module 06 — Invoices · spec §10.5 [LOCKED] · v1.0.0. The invoice is a presentation/grouping document over already-completed cases — it NEVER posts money (the sale posted at case completion §10.4; balance-forward accounts, payments lump-sum §11.5). Status Preparing → Issued → Cancelled (codes draft/issued/void — “Void” banned from UI); number assigned atomically on Issue (CEN01-INV-D02-2026-000045); issuing flags cases invoiced = duplicate-billing lock (Unique case_id across non-cancelled invoices); cancelling releases them. Body = one line per case, grouped by patient, with a summary/detailed print toggle (detailed expands teeth & materials from case_materials). Payer = doctor OR Unknown-Doctor patient (identical logic on patient_transactions). v1.0.1 (Ahmed 2026-07-21): money-copy audit — money arrives ONLY as lump-sum payments on the party account and is never related to a specific invoice or case: removed “quote the invoice No when paying” from all print footers, “payment for this bill” from the issued bar, and “oldest-first” phrasing from invoice surfaces. Built to the binding style guide: table default + grid toggle · Created column · teal-tinted bold headers + zebra · colored row actions only where legal · global pagination 100/200/500 · unified form components (colored icon-chip labels, 44px/2px inputs, indigo focus) · print options asked first (paper + layout) · size-aware print layouts + overflow-proof rules (DOC-12b) · accent role separation. Coherent example shared with the frozen Doctors statement: cases 0231/0219/0224/0228 — gross 9,889 − discounts 989 = EGP 8,900 everywhere. Permissions: create/issue/print = Receptionist+; Cancel = Owner/Manager/Accountant (§6.5).
